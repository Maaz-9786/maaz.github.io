Ticketing System in C

Project Overview

Developed an Account Ticketing System as an academic project, extending Assignment 1 Milestone 4. The system introduces a comprehensive ticketing component, enhancing data management and user interactions. The goal was to create a versatile tool for managing accounts, tickets, and messages within a secure and efficient framework.



Technologies Used

C Programming Language:

Leveraged the power of C for system-level programming, ensuring robust and efficient code.

Visual Studio:

Employed Visual Studio as the integrated development environment for streamlined project development.

GitHub:

Utilized Git and GitHub for version control, fostering collaboration and tracking changes.



Challenges

Precision in Automation: Achieving precision in automating data entry tasks using PyAutoGUI posed a challenge. Ensuring accurate navigation, data entry, and submission required careful consideration of timing and synchronization.


Error Handling: Robust error handling mechanisms were crucial to manage unexpected scenarios, such as changes in the form layout or network latency during the automation process. Implementing effective error handling was essential for maintaining the reliability of the tool.


Integration of Tkinter with Automation: Integrating the Tkinter GUI with the PyAutoGUI automation script required thoughtful design to synchronize the data entry process and maintain a seamless user experience. Balancing GUI responsiveness with automation was a significant aspect of the challenge.


Optimizing Data Storage: Efficiently storing and retrieving data in CSV Excel files using pandas required optimization to handle large datasets. Balancing performance considerations with resource efficiency was a key challenge in designing the data storage component.
